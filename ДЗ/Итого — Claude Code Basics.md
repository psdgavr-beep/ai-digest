## Реализация по урокам

### Урок 1 — База ✅

**По программе:**
- CLAUDE.md написан вручную (не через `/init`) — структура Why/What/How
- Настроен `.claude/settings.json` с разрешениями: allow (git, npm), deny (rm -rf, push --force, .env)
- 4-уровневая иерархия: глобальный `~/.claude/CLAUDE.md` + проектный

**Факт:**
```
~/.claude/CLAUDE.md        — стиль общения, язык, правила работы
ai-digest/CLAUDE.md        — стек, архитектура, команды, правила блога
ai-digest/.claude/rules/   — blog-style.md (вынесены правила стиля)
```

**Что хорошо:** CLAUDE.md компактный, правила стиля вынесены в rules/ — Progressive Disclosure применена правильно.

### Урок 4 — Контекст ✅


**Что хорошо:** Auto-memory реально работает — при начале новых сессий контекст подхватывается без пересказа.

### Урок 5 — MCP ✅

**По программе:** Model Context Protocol, Tavily, .mcp.json в проекте, Enable Tool Search.

**Факт:**
```json
// .mcp.json
{
  "mcpServers": {
    "tavily-search": {
      "command": "npx",
      "args": ["-y", "tavily-mcp@0.1.4"],
      "env": { "TAVILY_API_KEY": "..." }
    }
  }
}
```

Tavily используется субагентом news-scout для поиска тем. Ключ вынесен в .mcp.json (не в .env, что можно улучшить).

**Что сделали иначе:** Вместо Replicate (платный) для обложек использовали **Pollinations.ai** — бесплатно, без API-ключа, модель flux. Результат аналогичный.

**Что пропустили:** Context7 MCP, NotebookLM MCP — не понадобились для данного проекта.

### Урок 6 — Skills + Hooks ✅

**По программе:** Структура `.claude/skills/`, frontmatter, Context Fork, хуки на 26 событий, exit code 2, TDD.

**Факт — навыки:**
```
.claude/skills/
  hello/SKILL.md    — дата + последние 3 коммита
  cover/SKILL.md    — обложка через Pollinations.ai
  digest/SKILL.md   — полный цикл публикации
```

**Факт — хуки:**
```
.claude/hooks/validate-frontmatter.ps1   — Stop-хук
```
Хук проверяет frontmatter в изменённых `.md` файлах при каждой остановке Claude. Возвращает `exit 2` если отсутствуют обязательные поля (title, description, pubDate, tags).

В Stop-хуке также: сохраняет `git log -10` в файл автопамяти.

**Что пропустили:** TDD (Red→Green→Refactor) — не применяли, т.к. проект не требует тестирования кода.

### Урок 7 — Subagents + Agent Teams ✅

**По программе:** Субагенты с изолированным контекстом, .claude/agents/*.md, max_turns, Agent Teams через `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1`.

**Факт — 4 субагента:**

| Агент | Модель | Задача |
|-------|--------|--------|
| `news-scout` | Haiku | Ищет 5 тем через Tavily, сохраняет в `.claude/news-ideas.md` |
| `article-writer` | Sonnet | Пишет статью 400-500 слов + генерирует обложку |
| `article-judge` | Haiku | LLM-as-Judge: проверяет по правилам блога |
| `publisher` | Haiku | git add + commit + push |

Agent Teams включены: `CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS=1` в `.claude/settings.json`.

**Что хорошо:** Правильно подобраны модели — тяжёлые задачи на Sonnet, дешёвые проверки на Haiku.

**Что пропустили:** Worktree-изоляция (не нужна — агенты работают последовательно, не параллельно).

### Урок 8 — Agent Harness ✅

**По программе:** Harness engineering, LLM-as-Judge, верификация, headless, /loop.

**Факт — пайплайн /digest:**
```
/digest [тема]
  ↓ Шаг 0: если тема пустая — берёт из .claude/news-ideas.md
  ↓ Шаг 1: article-writer → статья + обложка
  ↓ Шаг 2: article-judge → проверка правил блога
      ✅ ПРОШЛА → Шаг 3
      ❌ НАРУШЕНИЯ → стоп, показать список
  ↓ Шаг 3: publisher → git push → Vercel
```

**LLM-as-Judge (article-judge) проверяет:**
- Объём 400-500 слов
- Запрещённые слова: инновационный, передовой, современный
- Упоминания производителей и моделей
- Frontmatter: 4 обязательных поля

**Тест в продакшне:** Судья поймал нарушение на первом прогоне ("современных" в тексте) → остановил публикацию → исправили → прошло.

**Сделано СВЕРХ программы:** `/schedule` через облако Anthropic (Антон упоминал как "на горизонте"). Три задания запланированы на 13-15 мая 2026, 10:00 МСК — работают без открытого VS Code.
